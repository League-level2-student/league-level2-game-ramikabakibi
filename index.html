Rocket Run

by: Rami

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I'm not exactly sure why I chose to make this game. I originally had another idea, but I thought it was too complicated and boring. The subject of my game just came to me and I decided to make it! I'm also very interested in astronomy, so that played a part in it.

2. What I Learned

My teacher suggested that I should mention when my character died why the character died, because sometimes it is not obvious why you died. The fuel bar ticks down silently, so sometimes it will run out, and you will die and not know why you died. Another classmate suggested decreasing the collision box size since sometimes the rocket would be just out of range of the asteroid but would still die. My teacher also suggested drawing the collision box when you activate the shield so that it looks more like you have a shield.

3. Architecture of My Game

I have a lot of classes, some dedicated to different objects in the game, like the asteroids, fuel tanks, and rocket, and some classes like GamePanel and ObjectManager that manage the objects and switch through states, run timers, draw the states, ect... The basic architecture of the game is that you are a rocket in outer space, and you use your arrow keys to avoid the asteroids, while also trying to capture fuel tanks to maintain your fuel. The different classes for objects are all managed in the object Manager, and array lists are used to add objects, where they are drawn in GamePanel.

4. Technical Implementation

To really implement all the aspects of my game that I wanted to, I relied a lot on timers. I had one overall timer that kept track of how long the game was running, and individual timers that managed the different objects, like the rocket, speed boosts, asteroids, and fuel tanks. Every time the timer would go off, the object would be added to the array list, and drawn on the screen. I used collision boxes as well to tell when for example the rocket collided with the asteroid, or the fuel tanks. One aspect of the game that I am proud of is my shield aspect and fuel aspects, and the aesthetics of the project as well. I used timers to have the shield bar go steadily up and the fuel steadily down, and had a lot of if statements, and variables so that when the shield got to 100, the user could press enter, and then the rocket would be invincible for 10 seconds or so, and the bar would go down until it recharged.

5. Most Challenging Part

The most challenging part of this project was really trying to get all of the different objects and timers to work alongside each other. I needed to have a lot of timers to achieve this, and trying to sort them out was pretty difficult. To overcome this, I tried to organize my code better and simplify it where I could.

6. Future Fixes and Updates

I would add why the user died when they died, and also would try to make the background less glitchy when it moved down by finding an image with more symmetry. I would also have stages in the game where the asteroids would fall more quickly as the game progressed and would add more features like bullets for the rocket to shoot.
